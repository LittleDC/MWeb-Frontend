迭代器与生成器

ES6新增
“迭代” 按照顺序反复多次执行一段程序
ES5新增Array.prototype.forEach()方法，向通用迭代需求迈进了一步

迭代器模式

可以把有些结构称为“可迭代对象”(iterable)，因为它们实现了正式的Iterable接口，而且可以通过迭代器Iterator消费。

迭代器iterator是按需创建的一次性对象，每个迭代器都会关联一个可迭代对象

可迭代协议 --- 实现Iterable接口

暴露一个属性作为“默认迭代器”，该属性的键为Symbol.iterator，该默认迭代器属性必须引用一个迭代器工厂函数，调用这个工厂函数必须返回一个新迭代器。

实现了Iterable接口的内置类型：

  1. 字符串 String
  2. 数组 Array
  3. 映射 Map
  4. 集合 Set
  5. arguments对象
  6. NodeList等DOM集合对象

接收可迭代对象的原生语言特性有：（不需要显示调用Iterable接口）

  1. for-of循环
  2. 数组解构
  3. 扩展运算符
  4. Array.from()
  5. 创建集合
  6. 创建映射
  7. Promise.all() / Promise.race()接收由期约组成的可迭代对象
  8. yield*操作符，在生成器中使用

迭代器协议

迭代器是一种一次性的对象，用于迭代与其关联的可迭代对象。

API：
必须：next()方法在可迭代对象中遍历数据。返回：IteratorReault对象（done & value）。
可选：return()方法指定在迭代器提前关闭时执行的逻辑，返回IteratorReault对象：
1. for-of循环通过break，continue，return或throw提前退出
2. 解构操作并未消费所有值

  • 数组的迭代器时不能关闭的（迭代器没有关闭，则还可以继续从上次离开的地方继续迭代

自定义迭代器

class Counter {
    constructor(limit) {
        this.limit = limit
    }
    
    [Symbol.iterator]() {
        let count = 1, limit = this.limit
        return {
            next() {
                if (count <= limit) {
                    return { done: false, value: count++ }
                } else {
                    return { done: true }
                }
            },
            return() {
                console.log('Exiting early')
                return { done: true }
            }
        }
    }
}

生成器

ES6新增
拥有在一个函数块内暂停和恢复代码执行的能力

生成器基础

函数名称前加一个星号（*）表示它是一个生成器。

调用生成函数会生成一个生成器对象，生成器对象一开始处于暂停执行状态(suspended)。
与迭代器相似，生成器也实现了Iterator接口，因此具有next()方法。调用该方法会使生成器开始或恢复执行
生成器只会在初次调用next()方法后开始执行。

yield关键字 --- 中断执行

类似函数的中间返回语句，它生成的值会出现在next()方法返回的对象里。
yield关键字只能在生成器函数内部使用，用在其他地方会抛出错误。必须直接位于生成器函数定义中。

  • yield作为函数的中间参数使用
    上一次生成器暂停的yield关键字会接收传给next()方法的第一个值。但第一次调用next()传入的值不会被使用，因为这一次调用是为了开始执行生成器函数
function* generatorFn(initial) {
    console.log(initial)
    console.log(yield)
    console.log(yield)
}

let g = generatorFn('foo') // 现在是处于暂停执行状态(suspended)

g.next('bar') // foo 第一次调用是为了开始执行生成器函数
g.next('baz') // baz
g.next('qux') // qux

提前终止生成器

  1. return() 强制生成器进入关闭状态。提供给return()方法的值，就是终止迭代器对象的值。
    for-of循环等内置语言结构会忽略状态为done:true的IteratorObject内部返回的值
  2. throw() 在暂停的时候将一个提供的错误注入到生成器对象中。如果错误未被处理，生成器就会关闭。
