创建代理 Proxy()

new Proxy(target, handler)
Proxy

捕获器 trap

每次在代理对象上调用这些基本操作时，代理可以在这些操作传播到目标对象之前先调用捕获器函数，从而拦截并修改相应的行为。

const target = {
	foo: 'bar'
}

const handler = {
   // 捕获器在处理程序对象中以方法名为键
	get () {
		return 'handler override'
	}
}

const proxy = new Proxy(target, handler)

console.log(target === proxy) //false
console.log(proxy.foo) // handler override

捕获器参数

目标对象，要查询的属性，代理对象

反射API --- Reflect

处理程序对象(handler)中所有可以捕获的方法都有对应的反射(Reflect)API方法。 -- 13个

const target = {
	foo: 'bar'
}

const handler = {
	get () {
		return Reflect.get(...arguments)
	}
}

const proxy = new Proxy(target, handler)

console.log(proxy.foo) // bar

捕获器不变式

防止捕获器定义出现过于反常的行为。
eg. 如果目标对象有一个不可配置且不可写的数据属性，那么在捕获器返回一个与该属性不同的值时，会抛出TypeError

可撤销代理

撤销代理操作是不可逆的。

const target = {
	foo: 'bar'
}

const handler = {
	get () {
		return 'itercetped'
	}
}

const { proxy, revoke } = Proxy.revocable(target, handler)

console.log(proxy.foo) // bar

revoke()

console.log(proxy.foo) //TypeError

实用反射API

状态标记:很多反射方法返回称作“状态标记”的布尔值，表示意图执行的操作是否成功。

以下反射方法提供只有通过操作符才能完成的操作：

  1. Reflect.get() 可以替代对象属性访问操作符
  2. Reflect.set() 可以替代'='赋值操作符
  3. Reflect.has() 可以替代'in'操作符或with()
  4. Reflect.deleteProperty() 可以替代delete操作符
  5. Reflect.construct() 可以替代'new'操作符

代理的问题与不足 --- this

代理中的this指向代理本身，不指向target

问题：代理Date类型--Date类型方法的执行依赖this值上的内部插位[[NumberDate]]

const target = new Date()
const proxy = new Proxy(target, {})

console.log(proxy instanceof Date) //true
proxy.getDate() //TypeError: 'this' is not a Date object
