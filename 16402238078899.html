⭐️期约与异步函数

ES6新增了正式的Promise(期约)应用类型，支持优雅的定义和组织异步逻辑。

异步编程

JS是单线程的

同步&异步

同步行为：对应内存中顺序执行的处理器指令
异步行为：类似于系统中断，即当前进程外部的实体可以触发代码执行。

以往的编程模式

  1. 异步返回值
  2. 失败处理
  3. 嵌套异步回调 --- 回调地狱

期约

期约（promise）是对尚不存在结果的一个替身。

期约基础

ES6新增的引用类型Promise，可以通过new操作符实例化。创建新期约时必须传入执行器(executor)函数作为参数。

let p = new Promise(() => {})
setTimeout(console.log, 0, p) // Promise <pending>

期约状态机

  • 待定（pending）
  • 兑现（fulfilled/resolved）
  • 拒绝（rejected）
  1. pending是期约的最初始状态。
  2. 在pending状态下，期约可以落定（settled） 为代表成功的fulfilled状态，或者代表失败的rejected状态。
  3. 落定是不可逆的。
  4. 期约的状态是私有的，不能直接通过JavaScript检测到。故意将异步行为封装起来，从而隔离外部的同步代码。

解决值、拒绝理由

每个期约只要状态切换为兑现，就会有一个私有的内部值（value）。
类似的，每个期约只要状态切换为拒绝，就会有一个私有的内部理由（reason）。

值和理由都是不可修改的引用

执行函数 --- 控制期约状态

期约的状态是私有的，所以只能在内部进行操作。内部操作在期约的执行器函数中完成。

执行器函数作用：

  1. 初始化期约的异步行为
    执行器函数是同步执行的。因为执行器函数是期约的初始化程序。
    hint:setTimeout执行是同步的，但其中的回调函数console.log是在异步队列中的
new Promise(() => setTimeout(console.log, 0, 'executor'))
console.log('sync test')
setTimeout(console.log, 1000, 'setTimeout test')
setTimeout(console.log, 0, 'promise initialized')

// sync test
// executor
// promise initialized
// (After 1 seconds) setTimeout test
  1. 控制状态的最终转换resolve()/reject()
  • 调用reject()会抛出错误
  • 无论resolve()和reject()中的哪个被调用，状态转换都不可撤销了，继续修改状态会静默失败（无效）。

Promise.resolve()

实例化一个解决的期约。 => 使该实例一开始便处于兑现状态

Promise.resolve()等价于new Promise((res, rej) => res())

  • Promise.resolve()如果传入的参数本身是一个期约，则该行为就类似一个空包装。 => 会保留传入期约的状态。可以说是幂等方法：执行多次产生的影响与执行一次相同。
let p = new Promise(() => {})
setTimeout(console.log, 0, p) // Promise <pending>
setTimeout(console.log, 0, Promise.resolve(p)) // Promise <pending>

setTimeout(console.log, 0, p === Promise.resolve(p)) // true

Promise.reject()

类似Promise.resolve()，实例化一个拒绝的期约并抛出一个异步错误。（该错误不能通过try/catch捕获，而只能通过拒绝处理程序捕获）

  • Promise.reject()不是幂等逻辑。如果传一个期约对象，则这个期约对象会成为它返回拒绝期约的理由。
setTimeout(console.log, 0, Promise.reject(Promise.resolve()))
// Promise {status: "rejected", result: Promise}
// error: Unhandled Promise Rejection: [object Promise]

同步/异步执行的二元性

期约真正的异步特性：是同步对象（在同步执行模式中使用），但也是异步执行模式的媒介。

try {
    throw new Error('foo')
} catch(e) {
    console.log(e) // Error: foo
}

try {
    Promise.reject(new Error('foo'))
} catch(e) {
    console.log(e)
}
// Unhandled Promise Rejection: Error: foo

第二个try/catch（同步代码）没有捕获到期约抛出的错误是因为它没有通过异步模式捕获错误。
